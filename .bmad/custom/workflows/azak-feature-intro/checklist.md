# Azak 기능 소개 블로그 생성 체크리스트

생성된 기능 소개 블로그를 검토하기 위한 체크리스트입니다.

## 필수 요소 확인

### 제목
- [ ] 기능 소개임이 명확한가?
- [ ] 핵심 가치가 포함되었는가?
- [ ] 친근하면서도 전문적인 톤인가?

### 기능 개요 섹션
- [ ] 이 기능이 무엇인지 명확히 설명했는가?
- [ ] 어떤 가치를 제공하는지 포함되었는가?
- [ ] 누가 사용하는지 언급되었는가?
- [ ] 배경 설명이 자연스럽게 녹아있는가? (있는 경우)

### 주요 기능 섹션
- [ ] 각 주요 기능이 상세히 설명되었는가?
- [ ] 무엇을, 어떻게, 왜 유용한지 포함되었는가?
- [ ] 화면 캡처가 적절히 삽입되었는가?

### 사용 방법 섹션 (핵심!)
- [ ] 단계별로 나누어져 있는가? (Step 1, 2, 3...)
- [ ] 각 단계마다 화면 캡처가 포함되었는가?
- [ ] 사용자 관점에서 작성되었는가?
- [ ] 이미지 경로가 상대 경로로 작성되었는가?
- [ ] 시작부터 결과까지 흐름이 명확한가?

### 기술 스택 섹션
- [ ] 사용된 기술들이 나열되었는가?
- [ ] 각 기술에 대한 간단한 설명이 있는가?
- [ ] 코드 예시가 필요한 경우 포함되었는가? (선택적)
- [ ] 코드가 간단하고 이해하기 쉬운가?

### 향후 계획 섹션 (선택적)
- [ ] 추가 예정 기능이 언급되었는가? (있는 경우)
- [ ] 긍정적이고 기대감을 조성하는 톤인가?

## 화면 캡처 확인

### 이미지 품질
- [ ] 모든 화면이 캡처되었는가?
- [ ] 이미지가 선명한가?
- [ ] 필요한 UI 요소가 모두 보이는가?
- [ ] 민감 정보가 포함되지 않았는가?

### 이미지 배치
- [ ] 각 이미지가 적절한 위치에 삽입되었는가?
- [ ] 이미지와 설명이 일치하는가?
- [ ] 사용 흐름에 맞게 순서대로 배치되었는가?

### 이미지 경로
- [ ] 상대 경로로 작성되었는가? (../../.playwright-mcp/...)
- [ ] 파일명이 올바른가? (feature-{{feature_slug}}-*.png)

## 톤&매너 확인

### 피해야 할 것
- [ ] 마침표(.) 과다 사용하지 않았는가?
- [ ] 과도한 격식체를 사용하지 않았는가?
- [ ] 불릿 포인트만 나열하지 않았는가?
- [ ] 이모지를 과하게 사용하지 않았는가?

### 사용해야 할 것
- [ ] "~해요", "~할 수 있어요" 종결어미 사용했는가?
- [ ] 친근한 설명 톤을 유지했는가?
- [ ] 사용자 관점에서 작성했는가?
- [ ] 간결하고 명확한 문장인가?

## 개발 과정 블로그와의 차이점

이 체크리스트는 **기능 소개 블로그**용입니다.

**개발 과정 블로그** (azak-dev-blog-generator)와의 차이:
- ❌ 문제 상황 섹션 없음 (선택적으로 개요에 포함)
- ❌ 해결 과정 섹션 없음
- ❌ 삽질 경험 없음
- ✅ 사용 방법이 핵심
- ✅ 화면 캡처가 더 많음
- ✅ 코드는 선택적
