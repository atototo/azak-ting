# Azak 개발 블로그 생성 체크리스트

생성된 블로그 포스트를 검토하기 위한 체크리스트입니다.

## 필수 요소 확인

### 제목
- [ ] 개발자스러운 톤으로 작성되었는가?
- [ ] 흥미를 유발하는 요소가 포함되었는가?
- [ ] 너무 딱딱하지 않은가?

### 문제 상황 섹션
- [ ] 어떤 문제가 있었는지 명확히 설명했는가?
- [ ] 왜 해결해야 했는지 이유가 포함되었는가?
- [ ] 구어체 톤이 적절히 사용되었는가?

### 해결 과정 섹션
- [ ] 코드 스니펫이 포함되었는가?
- [ ] 코드에 주석/설명이 추가되었는가?
- [ ] 언어 지정 (```python, ```typescript 등)이 되었는가?
- [ ] 민감 정보 (API 키, 토큰 등)가 제거되었는가?
- [ ] Before/After 비교가 있는가? (선택사항)
- [ ] 트러블슈팅 경험이 포함되었는가?

### 결과 섹션
- [ ] 화면 캡처 이미지가 포함되었는가?
- [ ] 이미지 경로가 상대 경로로 작성되었는가? (../../.playwright-mcp/...)
- [ ] 성능 개선 수치가 포함되었는가? (있는 경우)
- [ ] 최종 결과물이 명확히 설명되었는가?

### 배운 점 섹션
- [ ] 반말 구어체로 작성되었는가?
- [ ] 기술적 인사이트가 포함되었는가?
- [ ] 솔직한 감상이 들어갔는가?
- [ ] 다음 개선 방향이 언급되었는가?

### 기술 스택 섹션
- [ ] 사용한 기술들이 나열되었는가?
- [ ] 각 기술에 대한 간단한 설명이 있는가?

## 톤&매너 확인

### 피해야 할 것
- [ ] 마침표(.) 과다 사용하지 않았는가?
- [ ] 격식체 (입니다/습니다) 연발하지 않았는가?
- [ ] 불릿 포인트만 나열하지 않았는가?
- [ ] 이모지를 과하게 사용하지 않았는가? (섹션당 0~1개)
- [ ] 기술 용어를 무분별하게 나열하지 않았는가?

### 사용해야 할 것
- [ ] 구어체 종결어미 (~거든, ~더라, ~지 뭐) 사용했는가?
- [ ] 접속사 (근데, 그래서, 일단) 자연스럽게 사용했는가?
- [ ] 짧은 문장으로 작성했는가?
- [ ] 솔직한 감정 표현이 있는가?

## 기술적 정확성

### 코드
- [ ] 코드가 실제로 동작하는가?
- [ ] 문법 오류가 없는가?
- [ ] 보안 취약점이 없는가?
- [ ] 민감 정보가 완전히 제거되었는가?

### 설명
- [ ] 기술적 설명이 정확한가?
- [ ] 오해의 소지가 있는 표현이 없는가?
- [ ] 독자가 이해하기 쉽게 작성되었는가?

## 최종 확인

### 파일
- [ ] 파일명이 올바른가? (dev-{{topic_slug}}-{{date}}.md)
- [ ] 마크다운 문법이 올바른가?
- [ ] AI 생성 표기가 포함되지 않았는가?

### 이미지
- [ ] 모든 이미지가 캡처되었는가?
- [ ] 이미지 경로가 올바른가?
- [ ] 이미지가 적절한 위치에 삽입되었는가?

### 전체적인 흐름
- [ ] 문제 → 해결 → 결과 → 배운 점의 흐름이 자연스러운가?
- [ ] 독자가 따라가기 쉬운가?
- [ ] 개발 경험을 잘 전달하고 있는가?
