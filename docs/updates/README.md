## 문서 구조

각 업데이트 문서는 다음 형식을 따릅니다:

```
YYYY-MM-DD-{feature-name}.md
```

예시:
- `2025-11-21-reasoning-model-support.md`
- `2025-12-01-new-feature.md`

## 문서 작성 가이드

### 필수 섹션

각 업데이트 문서는 다음 섹션을 포함해야 합니다:

1. **변경 개요**: 변경 사항 요약
2. **AS-IS**: 변경 전 상태
3. **변경 필요 사유**: 왜 변경이 필요한가
4. **TO-BE**: 변경 후 상태
5. **변경 사항 상세**: 구체적인 변경 내용
6. **테스트 결과**: 변경 사항 검증
7. **사용 방법**: 사용자 가이드
8. **참고 사항**: 주의사항 및 추가 정보

### 템플릿

```markdown
# {Feature Name}

**작업 일자**: YYYY-MM-DD
**작업자**: {Team/Name}
**관련 이슈**: {Issue Link}

---
```
